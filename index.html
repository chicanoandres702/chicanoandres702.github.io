1. Introduction

The GitHub Importer is a desktop application that automates the process of creating milestones and issues in GitHub repositories. This tool streamlines project setup by importing data from a JSON file, which allows you to define milestones, associated issues, and all of the related information such as tasks, descriptions, overviews, labels, assignees, and comments.

2. Getting Started

2.1. Prerequisites

  • Python 3: Ensure you have Python 3.7 or later installed on your system. You can download it from the official Python website.
  • GitHub Account: You must have an active GitHub account to use this application.
  • GitHub Client ID and Secret:
    1. Create a Github app with a redirect URL of `http://localhost:8000/callback`.
    2. Copy the Client ID and Secret.
  • .env file: Create a .env file in the same folder as the main.py file and add your GitHub Client ID and Secret. 
    GITHUB_CLIENT_ID=
GITHUB_CLIENT_SECRET=
  • Installed Dependencies: Install all the required dependencies by running pip install -r requirements.txt where requirements.txt includes the libraries such as requests, flask, python-dotenv, tkinter.

2.2. Installation

  1. Clone Repository: 
    git clone 
cd
  2. Create Virtual Environment: 
    python3 -m venv venv
  3. Activate Virtual Environment: 
    source venv/bin/activate

    or 

    venv\Scripts\activate
  4. Install dependencies: 
    pip install -r requirements.txt

2.3. Running the Application

  1. Run main.py : Navigate to the project directory and execute python main.py
  2. The main application window will open up.

3. Using the Application

3.1. Authorization

  1. Click on the "Authorize with GitHub" button.
  2. This will open your default web browser and take you to GitHub to log in and grant permission to the GitHub Importer.
  3. Once authorized, you'll be redirected back to the application automatically. The application will retrieve the access token and will not require you to manually enter the code.

3.2. Repository Information

  1. Enter the repository owner (username or organization) in the "Repository Owner" field.
  2. Enter the name of the repository where you want to import the data in the "Repository Name" field.
  3. Make sure this is the repository you intend to update or you could cause errors with your repository data.

3.3. Import File

  1. Click the "Browse" button to select a JSON file that contains milestone and issue data.

3.4. Import Data

  1. Click the "Import Data" button.
  2. The application will begin creating milestones and issues in your selected GitHub repository, based on the structure of the provided JSON file.
  3. You'll see a status update message at the bottom of the app as the process is occurring. If an issue occurs, it will display here.
  4. The logs are written to the console. You can check there for more detailed information on each request.

4. JSON Input File Structure

The JSON file must have a specific structure, as shown below:


[
    {
        "title": "Sprint 1",
        "description": "This is the first sprint",
        "due_on": "2023-12-31T10:00:00Z",
        "state": "open",
         "issues": [
            {
                "title": "Implement User Authentication",
                "description": "Create user authentication functionality.",
                "overview": "A detailed overview of the user auth requirements.",
                "tasks": ["Implement login", "Implement logout"],
                "labels": ["bug", "enhancement"],
                "assignees": ["user1", "user2"],
                 "comments": [
                    "This is a first comment",
                     "This is a second comment"
                ]
            },
            {
                "title": "Design UI for Homepage",
                "description": "Create a mockup for the homepage design.",
                "overview": "A high-level overview of homepage.",
                "tasks": ["Create mockups", "Gather feedback"],
                "labels": ["design"],
                "assignees": ["user3"]
            }
        ]
    },
    {
        "title": "Sprint 2",
         "issues": [
            {
                "title": "Implement Database Migration",
                "description": "Migrate data from old database.",
                "overview": "Details on the migration.",
                 "tasks": ["Backup old DB", "Migrate", "Test"],
                 "labels": ["database"],
                "assignees": ["user1"]

            }
        ]
    }
]
  • The outermost structure should be an array of milestone objects.
  • Each milestone object can have the properties title, description, due_on, state and issues. due_on is the due date of the milestone, and state is the state the milestone should be set to.
  • Each milestone can have an array of issues inside the issues property.
  • Each issue object can have the properties: title, description, overview, tasks (as an array of strings), labels (as an array of strings), assignees (as an array of strings), and comments (as an array of strings).
  • Any additional properties are ignored.

5. Troubleshooting

  • Authorization Issues:
    • Ensure your GitHub account has the necessary permissions.
    • Make sure the credentials in the .env file are correct.
    • Double-check your GitHub application's settings to make sure you are using the correct URLs.
  • JSON Format Errors:
    • Make sure your JSON file adheres to the required structure.
    • You can use a JSON validator to find errors before importing the data.
  • API Rate Limiting:
    • GitHub's API is rate-limited. If the application encounters an error during a large import, it may be due to the rate limit being exceeded.
    • Wait for a while and try again, or limit the amount of data that you are importing at once.
  • Other Issues:
    • If the error message is not clear, check the logs outputted to the console.
    • If the application does not work correctly, verify you have followed the instructions in "Getting Started" correctly.
    • If you still encounter issues, create an issue on the GitHub repository.
